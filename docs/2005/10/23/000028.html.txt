<p>　　如果追随 Web2.0 网站的文化起源，我觉得 Google 就是原始模型。为什么叫 2.0？很简单，就是要区别于 1.0。什么 W3C、Ajax、RSS、Web API、社会性、开放性、用户组织内容……都是具体表现，实质就是玩不同于 1.0 的东西。只要以前没玩过，都能算 2.0。个人认为 Google 就是第一个玩 2.0 的：没有精心排版的表格嵌套页面，没有冗长的首页，没有专职编辑组织的内容，甚至感觉 Google 想让你在使用后快点离开。这些是最区别的，所以也是最可爱的。1.0 时，中国电信被称作 ISP(Internet Services Provider)，门户、包括不是门户的网站几乎都自称 ICP(Internet Content Provider)，貌似比 ISP 高级。Google 出现后，人们发现网站原来可以没有 Content，可以不是 ICP，而又成为 ISP(Internet Software Provider)。如今的 <a href="http://blog.donews.com/keso/archive/2005/08/26/526873.aspx">“Beta” 网站</a>，都是后 Google 时代的 ISP。</p>

<p>　　前两天装了 <a href="http://www.flock.com/">Flock</a>，感觉这种浏览器才是真正的 Next Generation，虽然还有很多问题。好比我觉得<a href="http://www.xdanger.com/2005/10/17/000023.html"> RSS 一定要等有好的在线阅读器才会好玩</a>，客户端的阅读器只能发掘它的表面应用；而 Web API 也要有了 Flock 这样的客户端才会好玩，<a href="http://www.marumushi.com/apps/flickrgraph/">flickr graph</a> 这样的也只是发掘了 Web API 的表面应用。</p>

<p>　　今后能真正像 Google 那样搞出名堂的，可能就是专注于做一个非常小的应用。因为有类似 Flock 的软件，所以任何将小功能发挥到极致的网站都是有很强生命力的。并且，因为小，用户才会觉得可爱，对手才会觉得不屑，Google 正成于此道（<a href="http://www.google.com/intl/zh-CN/corporate/tenthings.html">虽然现在反悔了</a>）。所以我比较看好现在做的最“小”的 <a href="http://del.icio.us/">del.icio.us</a>。一个网站，一个软件，一个功能，一个故事。</p>
